Visão Geral do Projeto

Entenda a arquitetura, princípios e stack tecnológica do FullStackHero

O que é o FullStackHero?

O FullStackHero .NET 10 Starter Kit é um framework modular e opinativo para construção de APIs SaaS multi-tenant e aplicações enterprise. Ele fornece uma base sólida com recursos prontos para produção, permitindo que equipes de desenvolvimento acelerem a entrega de projetos sem sacrificar qualidade arquitetural.

Filosofia: Production-first. Ao invés de começar do zero, você recebe módulos testados em batalha com Identity, Multitenancy, Auditing, caching, jobs, storage e observabilidade já configurados.

Arquitetura

Vertical Slice Architecture

O projeto adota Vertical Slice Architecture combinada com CQRS (Command Query Responsibility Segregation). Cada feature é auto-contida em seu próprio "slice" vertical: 

Features/v1/{Feature}/
├── {Feature}Command.cs      (ou Query)
├── {Feature}Handler.cs       (lógica de negócio)
├── {Feature}Validator.cs     (FluentValidation)
└── {Feature}Endpoint.cs      (Minimal API endpoint)

Camadas Principais

Building Blocks

Componentes reutilizáveis (Core, Persistence, Caching, Jobs, Mailing, Storage, Web)

Modules

Features modulares (Identity, Multitenancy, Auditing) que podem ser plugadas em qualquer API

Host (Playground)

Implementação de referência que consome os módulos e building blocks

Stack Tecnológica

Backend

.NET 10 - Framework principal

C# latest, nullable enabled, minimal APIs

Mediator - CQRS pattern

Commands/Queries desacoplados

FluentValidation - Validação

Regras de negócio declarativas

Mapster - Object mapping

Performance superior ao AutoMapper

Persistência de Dados

Entity Framework Core 10

ORM principal com domain events

PostgreSQL / SQL Server

Bancos suportados (Postgres default)

Specification Pattern

Queries reutilizáveis e testáveis

Finbuckle MultiTenant

Multi-tenancy com EF Core

Infraestrutura & DevOps

Redis - Distributed cache

Caching de alta performance

Hangfire - Background jobs

Jobs agendados e recorrentes

.NET Aspire - Orquestração

AppHost para dev/test com containers

Terraform - IaC

Deploy AWS (ECS, RDS, ElastiCache)

Observabilidade

OpenTelemetry

Traces, metrics e logs via OTLP

Serilog

Logging estruturado

Health Checks

Monitoramento de saúde da API

Auditing Module

Security, exception e activity audit

Princípios de Design

Domain-Driven Design (DDD)

Entities, Value Objects, Aggregates, Domain Events e Specifications para modelar o domínio de negócio

CQRS (Command Query Responsibility Segregation)

Separação clara entre comandos (escrita) e queries (leitura) usando Mediator pattern

Dependency Injection & IoC

Inversão de controle com Microsoft.Extensions.DependencyInjection para baixo acoplamento

Clean Architecture

Dependências apontam para dentro: Domain → Application → Infrastructure → Presentation

Estrutura de Diretórios 

src/
├── BuildingBlocks/          # Componentes reutilizáveis (NuGets)
│   ├── Core/                # DDD primitives, exceptions, abstractions
│   ├── Persistence/         # EF Core + specifications + migrations
│   ├── Caching/             # Redis distributed cache
│   ├── Jobs/                # Hangfire background jobs
│   ├── Mailing/             # MailKit email service
│   ├── Storage/             # File storage abstractions
│   ├── Web/                 # Auth, CORS, rate limiting, OpenAPI, OTel
│   └── Eventing/            # Event bus (RabbitMQ / InMemory)
│
├── Modules/                 # Features modulares (NuGets)
│   ├── Identity/            # ASP.NET Identity + JWT
│   │   ├── Features/v1/     # Endpoints organizados por feature
│   │   ├── Persistence/     # IdentityDbContext
│   │   └── Contracts/       # DTOs públicos
│   ├── Multitenancy/        # Finbuckle tenant management
│   └── Auditing/            # Security/exception/activity audit
│
├── Playground/              # Implementação de referência
│   ├── FSH.Playground.AppHost/   # Aspire orchestration
│   ├── Playground.Api/           # Host API
│   ├── Playground.Blazor/        # Blazor WASM UI
│   └── Migrations.PostgreSQL/    # Migrations consolidadas
│
└── Tests/
    ├── Architecture.Tests/  # NetArchTest rules
    ├── Identity.Tests/      # Unit tests Identity
    ├── Multitenancy.Tests/
    └── Auditing.Tests/
