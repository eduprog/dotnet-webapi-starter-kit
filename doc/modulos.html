Módulos

Features modulares: Identity, Multitenancy e Auditing

O que são Módulos?

Módulos são features auto-contidas que implementam funcionalidades específicas de negócio. Cada módulo é um pacote NuGet independente que pode ser "plugado" em qualquer API através da interface IModule.

Arquitetura Plugável: Cada módulo registra seus próprios serviços via ConfigureServices() e mapeia seus endpoints via MapEndpoints().

Interface IModule

public interface IModule
{
    string Name { get; }
    void ConfigureServices(IHostApplicationBuilder builder);
    void MapEndpoints(IEndpointRouteBuilder endpoints);
}

// Exemplo de uso no Program.cs
builder.AddModules(new[] {
    typeof(IdentityModule).Assembly,
    typeof(MultitenancyModule).Assembly,
    typeof(AuditingModule).Assembly
});

app.UseHeroPlatform(p => p.MapModules = true);

Identity Module

ASP.NET Identity + JWT Authentication

Módulo completo de autenticação e autorização baseado em ASP.NET Core Identity com emissão de JWT tokens.

Funcionalidades Principais

Token Generation

  • • Login com email/senha
  • • Geração de access token JWT
  • • Refresh tokens persistidos
  • • Rate limiting nos endpoints de auth

User Management

  • • Registro de usuários
  • • Confirmação de email
  • • Reset de senha
  • • Atualização de perfil

Roles & Permissions

  • • Gestão de roles (Admin, User, etc.)
  • • Permissions granulares
  • • Atribuição role ↔ user
  • • Claims customizados

Profile Images

  • • Upload de foto de perfil
  • • Storage abstraction (local/cloud)
  • • Servir imagens via endpoint
  • • Validação de tamanho/tipo

Exemplo: Login Endpoint

// POST /api/v1/tokens
public record GenerateTokenCommand(string Email, string Password) : ICommand<TokenResponse>;

public class GenerateTokenHandler : ICommandHandler<GenerateTokenCommand, TokenResponse>
{
    private readonly UserManager<ApplicationUser> _userManager;
    private readonly IJwtTokenGenerator _tokenGenerator;
    
    public async ValueTask<TokenResponse> Handle(GenerateTokenCommand command, CancellationToken ct)
    {
        var user = await _userManager.FindByEmailAsync(command.Email);
        if (user is null || !await _userManager.CheckPasswordAsync(user, command.Password))
        {
            throw new UnauthorizedException("Invalid credentials");
        }
        
        var token = await _tokenGenerator.GenerateTokenAsync(user);
        return new TokenResponse(token.AccessToken, token.RefreshToken, token.ExpiresIn);
    }
}

// Endpoint mapping
public static RouteHandlerBuilder MapGenerateTokenEndpoint(this IEndpointRouteBuilder endpoints)
{
    return endpoints.MapPost("/api/v1/tokens", async (GenerateTokenCommand command, IMediator mediator) =>
    {
        var result = await mediator.Send(command);
        return TypedResults.Ok(result);
    })
    .WithTags("Tokens")
    .AllowAnonymous()
    .WithRateLimit("auth-limit");  // 5 requests per minute
}

Multitenancy Module

Finbuckle MultiTenant para isolamento de dados

Módulo de multi-tenancy baseado em Finbuckle.MultiTenant que permite isolamento completo de dados entre tenants (clientes/organizações).

Características

Database per Tenant

Cada tenant tem seu próprio database schema isolado

Auto Migrations

Migrations aplicadas automaticamente na inicialização

Tenant Resolution

Resolução via header, subdomain ou route value

Provisioning

API para criar, ativar, desativar e deletar tenants

Como Funciona

  1. 1
    Request chega com tenant identifier

    Ex: Header X-Tenant: acme-corp

  2. 2
    Middleware resolve o TenantInfo

    Busca informações do tenant (connection string, configurações)

  3. 3
    DbContext é configurado dinamicamente

    Connection string aponta para o database do tenant

  4. 4
    Queries são isoladas automaticamente

    Dados de um tenant nunca vazam para outro

Endpoints Principais

// POST /api/v1/tenants - Criar novo tenant
{
  "identifier": "acme-corp",
  "name": "Acme Corporation",
  "email": "admin@acme.com",
  "connectionString": "Host=localhost;Database=fsh_acme;..."
}

// GET /api/v1/tenants/{identifier}/status - Status do tenant
{
  "identifier": "acme-corp",
  "isActive": true,
  "databaseExists": true,
  "pendingMigrations": []
}

// PUT /api/v1/tenants/{identifier}/activate - Ativar tenant
// PUT /api/v1/tenants/{identifier}/deactivate - Desativar tenant
// DELETE /api/v1/tenants/{identifier} - Deletar tenant

Configuração do DbContext

// DbContext multi-tenant
public class MyModuleDbContext : FshDbContext
{
    public MyModuleDbContext(DbContextOptions<MyModuleDbContext> options)
        : base(options)
    {
    }
    
    public DbSet<Product> Products => Set<Product>();
    
    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        
        // Configurações multi-tenant são herdadas de FshDbContext
        modelBuilder.Entity<Product>().IsMultiTenant();
    }
}

Migrações Multi-Tenant

O método UseHeroMultiTenantDatabases() garante que:

  • • Todos os tenants ativos tenham suas databases criadas
  • • Pending migrations sejam aplicadas automaticamente
  • • Seeders sejam executados se necessário

Auditing Module

Auditoria de segurança, exceções e atividades

Módulo de auditoria que registra eventos de segurança, exceções e atividades do usuário em um trail persistente e consultável.

Tipos de Auditoria

Security Audit

  • • Login attempts (success/fail)
  • • Logout events
  • • Password changes
  • • Role assignments

Exception Audit

  • • Unhandled exceptions
  • • Stack traces
  • • Request context
  • • User information

Activity Audit

  • • CRUD operations
  • • Business actions
  • • Data changes (before/after)
  • • Custom events

Estrutura do Audit Trail

public class AuditEntry
{
    public Guid Id { get; set; }
    public string UserId { get; set; }          // Quem fez
    public string Type { get; set; }            // Security/Exception/Activity
    public string Action { get; set; }          // Login, Create, Update, etc.
    public string Entity { get; set; }          // Entidade afetada
    public string EntityId { get; set; }        // ID da entidade
    public string OldValues { get; set; }       // JSON estado anterior
    public string NewValues { get; set; }       // JSON novo estado
    public string IpAddress { get; set; }       // IP do usuário
    public string UserAgent { get; set; }       // Browser/client
    public DateTime Timestamp { get; set; }     // Quando
    public string TenantId { get; set; }        // Multi-tenant aware
}

Registro Automático

// Auditoria automática via SaveChanges interceptor
protected override async Task<int> SaveChangesAsync(CancellationToken ct)
{
    // Detecta mudanças nas entidades
    var auditEntries = OnBeforeSaveChanges();
    
    var result = await base.SaveChangesAsync(ct);
    
    // Registra audit trail
    await OnAfterSaveChangesAsync(auditEntries, ct);
    
    return result;
}

// Auditoria manual
await _auditService.LogAsync(new AuditEntry
{
    Type = "Security",
    Action = "Login",
    UserId = user.Id,
    IpAddress = httpContext.Connection.RemoteIpAddress?.ToString()
});

Boas Práticas

  • Retenção: Configure políticas de retenção (ex: manter últimos 90 dias)
  • GDPR: Considere anonimizar/deletar audit logs quando usuário é removido
  • Performance: Audit logs em tabela separada com índices apropriados
  • Segurança: Logs de auditoria devem ser imutáveis (append-only)

Criando seu Próprio Módulo

Você pode criar seus próprios módulos seguindo o mesmo padrão: 

// 1. Implementar IModule
public class CatalogModule : IModule
{
    public string Name => "Catalog";
    
    public void ConfigureServices(IHostApplicationBuilder builder)
    {
        // Registrar DbContext, repositories, services...
        builder.Services.AddDbContext<CatalogDbContext>(...);
        builder.Services.AddScoped<IProductRepository, ProductRepository>();
    }
    
    public void MapEndpoints(IEndpointRouteBuilder endpoints)
    {
        // Registrar todos os endpoints do módulo
        endpoints.MapCreateProductEndpoint();
        endpoints.MapGetProductsEndpoint();
        endpoints.MapUpdateProductEndpoint();
    }
}

// 2. Criar features em Features/v1/
Features/v1/Products/
├── CreateProduct/
│   ├── CreateProductCommand.cs
│   ├── CreateProductHandler.cs
│   ├── CreateProductValidator.cs
│   └── CreateProductEndpoint.cs
└── GetProducts/
    └── ...

// 3. Adicionar ao Program.cs
builder.AddModules(new[] { typeof(CatalogModule).Assembly });
