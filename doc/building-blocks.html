Building Blocks

Componentes reutilizáveis que formam a fundação do framework

O que são Building Blocks?

Building Blocks são componentes reutilizáveis e agnósticos de domínio que fornecem funcionalidades fundamentais para qualquer aplicação enterprise. Eles são empacotados como NuGet packages individuais e podem ser utilizados independentemente.

Filosofia: Cada Building Block resolve um problema específico de infraestrutura (caching, persistência, jobs, etc.) e pode ser usado em qualquer projeto .NET, não apenas neste starter kit.

Exemplo: BaseEntity

public abstract class BaseEntity<TId>
{
    public TId Id { get; protected set; }
    
    private readonly List<IDomainEvent> _domainEvents = new();
    public IReadOnlyList<IDomainEvent> DomainEvents => _domainEvents.AsReadOnly();
    
    protected void RaiseDomainEvent(IDomainEvent domainEvent)
    {
        _domainEvents.Add(domainEvent);
    }
    
    public void ClearDomainEvents() => _domainEvents.Clear();
}

Persistence

Entity Framework Core + Specifications + Migrations

Funcionalidades

  • DbContext Base: FshDbContext com multi-tenancy, soft delete, auditing automático
  • Repository Pattern: IRepository<T> genérico com CRUD + Specifications
  • Specification Pattern: Queries reutilizáveis e testáveis
  • Domain Events Dispatching: Interceptor que dispara eventos após SaveChanges
  • Database Providers: PostgreSQL e SQL Server suportados

Exemplo: Specification Pattern

// Definir uma specification
public class ActiveUsersSpec : Specification<User>
{
    public ActiveUsersSpec()
    {
        Query.Where(u => u.IsActive == true)
             .OrderBy(u => u.CreatedOn);
    }
}

// Usar em um repository
var activeUsers = await repository
    .ListAsync(new ActiveUsersSpec(), cancellationToken);

// Specification com paginação
public class UsersByRoleSpec : Specification<User>
{
    public UsersByRoleSpec(string role, int page, int pageSize)
    {
        Query.Where(u => u.Role == role)
             .Skip((page - 1) * pageSize)
             .Take(pageSize);
    }
}

Exemplo: ICacheService

public interface ICacheService
{
    Task<T?> GetAsync<T>(string key, CancellationToken ct = default);
    Task SetAsync<T>(string key, T value, TimeSpan? expiration = null, CancellationToken ct = default);
    Task RemoveAsync(string key, CancellationToken ct = default);
    Task RemoveByPrefixAsync(string prefix, CancellationToken ct = default);
}

// Uso
var cachedUser = await _cacheService.GetAsync<User>($"user:{userId}");
if (cachedUser is null)
{
    cachedUser = await _repository.GetByIdAsync(userId);
    await _cacheService.SetAsync($"user:{userId}", cachedUser, TimeSpan.FromMinutes(30));
}
return cachedUser;

Jobs

Background jobs com Hangfire

Integração com Hangfire para jobs de background, agendados e recorrentes. Suporta PostgreSQL storage e dashboard de monitoramento.

Tipos de Jobs

Fire-and-Forget

Executados uma vez em background

Delayed

Agendados para executar após um delay

Recurring

Jobs recorrentes (CRON expressions)

Exemplo: Enfileirar Jobs

// Fire-and-forget
BackgroundJob.Enqueue<IEmailService>(x => x.SendEmailAsync(email));

// Delayed (executar daqui 1 hora)
BackgroundJob.Schedule<IReportService>(
    x => x.GenerateReportAsync(reportId),
    TimeSpan.FromHours(1));

// Recurring (todos os dias às 3am)
RecurringJob.AddOrUpdate<ICleanupService>(
    "cleanup-temp-files",
    x => x.CleanupTempFilesAsync(),
    "0 3 * * *");

Eventing

Sistema de eventos de integração com suporte a RabbitMQ e implementação in-memory para testes.

Padrões

Inbox Pattern

Garante processamento exactly-once de eventos recebidos

Outbox Pattern

Garante publicação transacional de eventos

Exemplo: Publicar e Consumir Eventos

// Definir evento
public record UserRegisteredEvent(Guid UserId, string Email) : IIntegrationEvent;

// Publicar
await _eventBus.PublishAsync(new UserRegisteredEvent(user.Id, user.Email));

// Handler
public class UserRegisteredEventHandler : IIntegrationEventHandler<UserRegisteredEvent>
{
    public async Task HandleAsync(UserRegisteredEvent @event, CancellationToken ct)
    {
        // Enviar email de boas-vindas, etc.
        await _mailService.SendWelcomeEmailAsync(@event.Email, ct);
    }
}
